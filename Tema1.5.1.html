1.5.1 Pasos para la metodología de la Investigación de Operaciones.

1.- Formulación del problema

La primera actividad que se debe realizar es el estudio del sistema y el desarrollo de un resumen bien definido del problema que se va a analizar. Esto incluye determinar los objetivos apropiados, las restricciones sobre lo que se puede hacer, las interrelaciones del área bajo estudio con otras áreas de la organización, los diferentes cursos de acción posibles y las variables de decisión.

2.- Construcción del modelo matemático para representar el sistema bajo estudio

Los modelos matemáticos generalmente incluyen dos tipos de ecuaciones: la ecuación de la función objetivo y las ecuaciones de las restricciones, las cuales incluyen a variables controlables (de decisión) y variables incontrolables, expresándose como el problema de elegir los valores de las variables de decisión de manera que se maximice o minimice la función objetivo, sujeta a las restricciones dadas.

3.- Obtención de una solución a partir del modelo

El siguiente paso es resolver el modelo, es decir, obtener los mejores valores para la variable de decisión, es decir, aquellos valores que satisfacen simultáneamente todas las limitaciones y proporciona el mejor valor para la función objetivo.

4.- Validación del modelo

Después de resolver el modelo matemático, es extremadamente importante validar la solución, es decir, revisar la solución cuidadosamente para ver que los valores tengan sentido y las decisiones resultantes pueden llevarse a cabo. Algunas de las razones para hacer esto son:

  1. El modelo matemático puede no haber captado el problema real.
  2. Ciertos aspectos del problema pueden haberse pasado por alto, omitido deliberadamente o simplificado.
  3. Los datos pueden haberse estimado o registrado incorrectamente.

En general, si la solución no puede llevarse a cabo, hay que modificar el modelo para reflejar más exactamente las limitaciones del problema real (y obtener una nueva solución). También es importante darse cuenta de que aún cuando el modelo y la solución pueden ser válidos, tal vez no sea factible llevar a cabo una decisión basándose en esos resultados.

5.- Implantación

Esta etapa es crítica, es aquí donde se cosecharán los beneficios del estudio. Por lo tanto, es importante que el equipo de I.O. participe, tanto para asegurar que las soluciones del modelo se traduzcan con exactitud a un procedimiento operativo, como para corregir cualquier defecto en la solución que salga a la luz en ese momento.

Primero, el equipo de I.O. da una cuidadosa explicación a la gerencia operativa sobre el nuevo sistema que se va adoptar y su relación con la realidad operativa. Enseguida, estos dos grupos comparten la responsabilidad de desarrollar los procedimientos requeridos para poner este sistema en operación y la gerencia operativa capacita al personal que participa y se inicia entonces el nuevo curso de acción. Si tiene éxito, el nuevo sistema se podrá emplear durante algunos años [9].
